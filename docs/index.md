# Docs Index

This is the top-level discovery page for project context.

Everything in `docs/` should either be linked here directly or be reachable from one of the section indexes below.

## Sections

- [`README.md`](./README.md): overview of the documentation system and organization rules.
- [`knowledge/README.md`](./knowledge/README.md): general knowledge, learnings, decisions, and ongoing context.
- [`deployment/README.md`](./deployment/README.md): deployment approach, runtime configuration, and operational notes.
- [`references/README.md`](./references/README.md): external references and design/source material.
- [`tasks/README.md`](./tasks/README.md): task tracking and execution notes.
- [`plans/README.md`](./plans/README.md): implementation plans and future work structure.

## Discoverability Rule

When adding a new document:

1. Place it in the closest matching section.
2. Link it from that section's README.
3. If it matters to more than one workflow, also link it from this index.
